Here is week 7 update for my project:

This week, I added support for alerts as part of [PR #29](https://github.com/MDAnalysis/mdadash/pull/29){:target="_blank"}.

Here are some of the changes made as part of this PR:

>
- Backend support for creating and managing alerts
- Frontend support for displaying alerts
- Command line option to clear alerts during startup
- Backend and Frontend tests for all new functionality

Here is an example screenshot of the newly added alerts page:

![Alerts](assets/img/gsoc-2026-week-7-alerts.png)
_Alerts_

As one of the actions that widgets can trigger at runtime, I added support for pause from widgets as part of [PR #27](https://github.com/MDAnalysis/mdadash/pull/27){:target="_blank"}. I made some minor name changes as discussed in the sync up meeting.

Here are some of the changes made as part of this PR:

>
- Allow widget analyses code to trigger pause event
- Rename per-frame to every-frame
- `_patch_IMDReader` is now a static method to not serialize WidgetManager
- Fix add widget layout issue

I created a new discussion to describe the current implementation of [Batching and Parallelization](https://github.com/MDAnalysis/mdadash/discussions/28){:target="_blank"} as it came up in the sync up meeting.

I added a minimal `dependabot.yml` to the project with a schedule for GitHub actions and merged associated PRs created by `dependabot`.

I created [PR #35](https://github.com/MDAnalysis/mdadash/pull/35){:target="_blank"} to add a widget for MSD Analysis based on the issue Amru raised to support lag-time based calculations in [Issue #34](https://github.com/MDAnalysis/mdadash/issues/34){:target="_blank"}. This PR is currently in review.

There was some minor cleanup of widget invocation I did as part of [PR #36](https://github.com/MDAnalysis/mdadash/pull/36){:target="_blank"}.

Here are some of the changes made as part of this PR:

>
- Remove passing redundant batch_size param
  - The buffer / batch size is available in trajectory
- Reset frame to the latest timestep before each widget invocation
  - Widget runs can iterate trajectory and hence required

Release [v0.0.6](https://github.com/MDAnalysis/mdadash/releases/tag/v0.0.6){:target="_blank"} has all the core functionality for the project.

Next week, I will start with the support for adding user-defined custom analysis widgets through the Notebook like interface from the dashboard UI.
